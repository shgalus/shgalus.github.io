26  * Solves the linear programming problem by revised simplex method.
27  * Solves the problem in standard form: minimize \f$c^T x\f$ subject
28  * to \f$Ax = b\f$, \f$x \geq 0\f$, where \f$b \geq 0\f$. \f$m \geq
29  * 1\f$ is the number of constraints, \f$n \geq 1\f$ is the number of
30  * variables, \f$A\f$ is an \f$m \times n\f$ matrix, \f$b\f$ is an
31  * \f$m \times 1\f$ vector, \f$c\f$ is an \f$n \times 1\f$ vector,
32  * \f$x\f$ is an \f$n \times 1\f$ vector. The parameter \a eps defines
33  * the tolerance: for a real number a, if abs(a) < eps, then a is
34  * treated as 0.
35  *
36  * \retval 0 ok, solution found
37  * \retval 1 invalid input
38  * \retval 2 no feasible solution
39  * \retval 3 unbounded solution
40  *
41  * If the returned value is 0, then \a x(j), j = 0, ..., n - 1,
42  * contains values of \a x and \a f is the value of objective function
43  * in \a x. If the return value is not 0, the values of f and x are
44  * not changed by this function.
45  *
46  * \date Written on 8 January 2012.
47  *
48  * \implementation Translated from \cite syslo-deo-kowalik-1999, p.
49  * 23-25.
50  */

110 /**
111  * Wolfe method for quadratic programming problem.
112  *
113  * Minimizes \f$px + x^TCx\f$ subject to \f$Ax = b\f$, \f$x \geq 0\f$.
114  * \f$p\f$ and \f$x\f$ are \f$n \times 1\f$ vectors, \f$b\f$ is an
115  * \f$m \times 1\f$ vector, \f$A\f$ is an \f$m \times n\f$ matrix,
116  * \f$C\f$ is an \f$n \times n\f$ matrix. The matrix \f$C\f$ must be
117  * symmetric (only upper right triangle is used) and should be
118  * positive definite. The number of linear equality constraints
119  * \f$m\f$ and the number of variables \f$n\f$ must be greater than 0.
120  *
121  * \retval 0 ok, solution found
122  * \retval 1 invalid arguments
123  * \retval 2 not enough memory
124  * \retval 3 the maximum number of simplex iteration steps exceeded
125  * \retval 4 no feasible solution in initiation phase
126  * \retval 5 no feasible solution in recursion phase
127  * \retval 6 unbounded solution in initiation phase
128  * \retval 7 unbounded solution in recursion phase
129  *
130  * If the returned value is 0, \a x contains the solution, \a f
131  * contains the minimum value, otherwise they are udefined.
132  *
133  * \note The parameter \a C should be a vector of size n x (n + 1) / 2
134  * which contains the upper right triangle of the matrix \f$C\f$
135  * written by rows.
136  *
137  * The problem \f{align*} \mbox{minimize} & \quad -6x_1 + 2x_1^2 -
138  * 2x_1x_2 + 2x_2^2 \\ \mbox{subject to} & \quad x_1 + x_2 \leq 2, \\
139  * & \quad x_j \geq 0 \f} (\cite gass-1980, p. 285) can be solved by
140  * the following program.
141  *
142  * \example_program \include wolfe.cc
143  *
144  * The output means that \f$x_1 = 3/2\f$, \f$x_2 = 1/2\f$ and the
145  * minimum is \f$-11/2\f$.
146  *
147  * \date Written on 10 May 2009.
148  * \todo Check if the page 285 of the example is correct.
149  *
150  * \internal
151  * Implementation.
152  * ===============
153  *
154  * Problem.
155  * --------
156  *
157  * Let A be an m x n matrix, b >= 0 an m x 1 vector, p an 1 x n
158  * vector, C an n x n symmetric, positive semidefinite matrix, and x
159  * an n x 1 vector. The quadratic programming problem is:
160  *
161  * minimize px + x^(T)Cx
162  * subject to Ax = b,
163  * x >= 0.
164  *
165  * The Wolfe method is based on the theorem that x >= 0 solves the
166  * quadratic programming problem if Ax = b and there exist n x 1
167  * vector v >= 0 and m x 1 vector u such that
168  *
169  * v^(T)x = 0,
170  * 2Cx - v + A^(T)u + p^(T) = 0.
171  *
172  * Convergence of the "short form" of the method, used here, requires
173  * that either p = 0 or C is positive definite.
174  *
175  * Algorithm.
176  * ----------
177  *
178  * Let z1, z2 be n x 1 vectors and w be an m x 1 vector.
179  *
180  * 1. Initiation. Use the simplex method to minimize w(1) + ... + w(m)
181  * to zero subject to
182  *
183  * Ax + w = b,
184  * 2Cx - v + A^(T)u + z1 - z2 = -p^(T),
185  * x, v, z1, z2, w >= 0,
186  * keeping u and v zero. Discard w and the unused components of z1 and
187  * z2. Let the remaining n components be denoted by z, and their
188  * coefficients by E. We have a solution of the system
189  *
190  * Ax = b,
191  * 2Cx - v + A^(T)u + Ez = -p^(T),
192  * x, v, z >= 0.
193  *
194  * 2. Recursion. Take u = u1 - u2, u1, u2 >= 0. Use the simplex method
195  * to minimize z(1) + z(2) + ... + z(n) to zero subject to
196  *
197  * Ax = b,
198  * 2Cx - v + A^(T)u1 - A^(T)u2 + Ez = -p^(T),
199  * x, v, u1, u2, z >= 0,
200  * under the side condition: for k = 1, ..., n: if x(k) is in the
201  * basis, do not admit v(k); if v(k) is in the basis, do not admit
202  * x(k).
203  *
204  * 3. Termination. If p = 0 or C is positive definite, the objective
205  * function z(1) + ... + z(n) will vanish in at most 3n over n (Newton
206  * symbol) iterations, yielding z = 0. The x-part of the terminal
207  * basic solution is a solution of the quadratic programming problem.
208  *
209  * Implementation.
210  * ---------------
211  *
212  * The initial simplex table for the initiation phase is
213  *
214  * |Basis| x | v | u | z | w |RHS |
215  * |:---:|:-:|:-:|:-:|:-:|:-:|:--:|
216  * | w | A | 0 | 0 | 0 | I | b |
217  * | z | 2C| -I|A^T| I | 0 |-p^T|
218  * | | | 0 | 0 | 0 | 0 | |
219  *
220  * The n variables z are choosen to have -p^T >= 0. The same table may
221  * be used in the recursion phase. The numberes of basis variables
222  * will be kept in a special vector, so there is no need to keep their
223  * coefficients in the table. The table in both initiation and
224  * recursion will be
225  *
226  * |Basis| x | v | u |RHS |
227  * |:---:|:-:|:-:|:-:|:-: |
228  * | w | A | 0 | 0 | b |
229  * | z | 2C| -I|A^T|-p^T|
230  * | | | 0 | 0 | |
231  *
232  * In the initiation phase u and v will not enter the basis and in the
233  * recursion phase w will not enter the basis and the side condition
234  * will be used. Numbers of variables will be as follows:
235  *
236  * x(1) ... x(n) - 0 ... n - 1
237  * v(1) ... v(n) - n ... 2n - 1
238  * u(1) ... u(m) - 2n ... 2n + m - 1
239  * z(1) ... z(n) - 2n + m ... 3n + m - 1
240  * w(1) ... w(m) - 3n + m ... 3n + 2m - 1
241  *
242  * The simplex table will be kept in the matrix
243  *
244  * t(m + n + 1, 2n + m + 1),
245  *
246  * the m + n-th row and 2n + m-th column being added for coefficients.
247  * The numbers of basis and non-basis variables will be kept in the
248  * integer vectors B(m + n) and N(2n + m).
249  *
250  * Both phases will be performed within the same loop. The flow is
251  * arranged to affect whole rows of the table t in order to make
252  * eventual version using scratch file easier to write.
253  *
254  * References.
255  * -----------
256  *
257  * \cite wolfe-1959, \cite gass-1980, p. 276-282,
258  * \cite grabowski-1980, p. 245-251,
259  * \cite bronsztejn-siemiendiajew-musiol-muhlig-2004, p. 943-945
260  */
261 int wolfe(const Vecdouble& p,
262  const Vecdouble& C,
263  const Matdouble& A,
264  const Vecdouble& b,
265  Vecdouble& x,
266  double& f);
